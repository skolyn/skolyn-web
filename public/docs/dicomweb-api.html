DICOMweb API

1. Overview

DICOMweb is the DICOM standard for RESTful web services, replacing traditional DICOM networking for modern HTTP-based medical image exchange. Skolyn implements the full DICOMweb specification (DICOM PS3.18) including WADO-RS (Web Access to DICOM Objects - RESTful Services) for retrieving studies, series, and instances; STOW-RS (Store Over the Web - RESTful Services) for uploading DICOM objects; and QIDO-RS (Query based on ID for DICOM Objects - RESTful Services) for searching studies, series, and instances. All DICOMweb endpoints are served over HTTPS (TLS 1.3) with OAuth 2.0 Bearer token authentication, making integration straightforward for modern web applications, mobile apps, and cloud-native PACS systems.

4. WADO-RS (Retrieve)

4.1 Retrieve Study

# Retrieve all instances in a study
GET /dicomweb/studies/1.2.840.113619.2.12345
Authorization: Bearer <token>
Accept: multipart/related; type="application/dicom"

# Retrieve study metadata (JSON, no pixel data)
GET /dicomweb/studies/1.2.840.113619.2.12345/metadata
Accept: application/dicom+json

# Retrieve specific series
GET /dicomweb/studies/1.2.840.113619.2.12345/series/1.2.840.113619.2.67890
Accept: multipart/related; type="application/dicom"

# Retrieve specific instance
GET /dicomweb/studies/1.2.840.113619.2.12345/series/1.2.840.113619.2.67890/instances/1.2.840.113619.2.11111
Accept: application/dicom

4.2 Rendered Frame Retrieval

# Retrieve a rendered JPEG/PNG frame
GET /dicomweb/studies/{studyUID}/series/{seriesUID}/instances/{instanceUID}/frames/1/rendered
Accept: image/jpeg
Authorization: Bearer <token>

# Query parameters for rendering:
# ?window=center,width (e.g., ?window=40,400 for abdomen)
# ?quality=75 (JPEG quality 1-100)
# ?viewport=512,512 (resize to 512x512)

6. Skolyn AI Extensions

Skolyn extends the standard DICOMweb API with custom endpoints for accessing AI-specific functionality without requiring separate FHIR calls:

# Get AI analysis status for a study
GET /dicomweb/studies/{studyUID}/ai/status
Authorization: Bearer <token>

# Response
{
  "studyInstanceUID": "1.2.840.113619.2.12345",
  "processingStatus": "complete",
  "module": "rhenium",
  "subtype": "brain",
  "findingCount": 3,
  "criticalFindings": true,
  "processingTime": 18.4,
  "attentionMapAvailable": true
}

# Retrieve AI attention map overlay as PNG
GET /dicomweb/studies/{studyUID}/series/{seriesUID}/instances/{instanceUID}/ai/attention-map
Accept: image/png

7. Rate Limiting & Pagination

DICOMweb API requests are rate-limited per OAuth client: 500 requests/minute for queries, 100 requests/minute for stores, and 1,000 requests/minute for retrievals. Pagination is handled via limit and offset query parameters. The response includes custom headers for pagination state: X-Total-Count (total matching results), X-Limit, and X-Offset.
