Continuous Learning

Model Lifecycle Management: Federated Learning, Active Learning, Data Flywheel Architecture, and Regulatory-Compliant Model Updates
Version: 3.0
Last Updated: February 2026
Classification: Confidential
Owner: AI Research

1. Overview

Medical AI models face a fundamental challenge: the data distribution in clinical practice continuously evolves — new scanner hardware, changing patient demographics, emerging disease patterns, and evolving clinical protocols create dataset shift that can degrade model performance over time. Skolyn's Continuous Learning Framework (CLF) addresses this challenge through a systematic approach to model improvement that maintains regulatory compliance while enabling ongoing performance enhancement. The CLF encompasses four pillars: data flywheel (automated collection and curation of training data from production deployments), active learning (intelligent selection of the most informative cases for expert annotation), federated learning (multi-site model training without centralizing patient data), and controlled model updates (a gated release process that ensures every model update passes clinical validation before production deployment).

2. Data Flywheel Architecture

The data flywheel automatically captures learning opportunities from every study processed by Terbium OS. When a radiologist reviews an AI finding and provides feedback (confirm, dismiss, modify), this implicit supervision signal is captured and stored in the learning database. Explicit corrections — where a radiologist marks a finding missed by the AI or dismisses a false positive — are particularly valuable training signals. The flywheel pipeline de-identifies all captured data (removing PHI per HIPAA Safe Harbor), associates the de-identified image data with the radiologist's ground truth labels, and queues the labeled examples for inclusion in the next training cycle. Institutional data governance policies control whether local data enters the training pipeline — each institution retains full ownership and can opt out of data contribution while still receiving model updates trained on data from participating institutions.

3. Active Learning

Active learning intelligently selects the most informative cases for expert annotation, maximizing model improvement per unit of annotation effort. The active learning strategy combines three selection criteria: uncertainty sampling (studies where the model's prediction confidence is lowest, indicating decision boundary proximity), diversity sampling (studies that are maximally dissimilar from existing training data, improving representation coverage), and error sampling (studies flagged by clinicians as AI errors, directly targeting failure modes). The active learning coordinator presents selected cases to expert annotators via a web-based annotation interface with standardized labeling protocols, ensuring annotation quality and consistency.

4. Federated Learning

Federated learning enables multi-institutional model training without transferring patient data between sites. Each participating institution runs local model training on their de-identified data, producing model weight updates (gradients) that are encrypted and transmitted to the central aggregation server. The aggregation server combines updates from all participating sites using Federated Averaging (FedAvg) with differential privacy guarantees (epsilon=8, delta=10^-5) to produce an updated global model. The global model is then distributed back to all sites. This approach preserves data sovereignty — patient images never leave the originating institution — while enabling the model to learn from diverse multi-institutional data. Secure aggregation using homomorphic encryption ensures that individual site contributions cannot be reverse-engineered from the aggregated model.

6. Regulatory Compliance for Model Updates

The FDA's Predetermined Change Control Plan (PCPC) framework, established in the 2023 guidance on Marketing Submission Recommendations for a Predetermined Change Control Plan for Artificial Intelligence/Machine Learning-Enabled Device Software Functions, enables pre-authorized model modifications within specified bounds. Skolyn's PCPC defines: the types of modifications permitted (training data expansion, hyperparameter adjustments, architecture modifications within families), modification protocols (validation methodology, sample sizes, statistical criteria), and performance bounds (minimum acceptable sensitivity/specificity thresholds). Updates that fall within the PCPC scope can be deployed without new regulatory submissions, while modifications exceeding the PCPC scope trigger a supplemental submission process.
