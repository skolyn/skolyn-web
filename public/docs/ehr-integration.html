1. Overview

Integrating Skolyn with Electronic Health Record (EHR) systems bridges the gap between diagnostic AI findings and the clinician's primary workspace. This guide covers integration patterns for Epic, Cerner (Oracle Health), MEDITECH, Allscripts, and athenahealth, as well as generic FHIR-based integration for any EHR supporting the SMART on FHIR framework. Skolyn's EHR integration delivers AI findings directly into clinical notes, embeds interactive dashboards within the EHR via CDS Hooks, and surfaces critical finding alerts in the clinician's inbox. The goal is zero-context-switch access to AI diagnostic insights — radiologists and referring physicians should never need to leave their EHR to review Skolyn's analysis.

2. Integration Patterns

2.1 CDS Hooks (Clinical Decision Support)

Skolyn implements the CDS Hooks specification (v1.1) to provide AI-driven clinical decision support triggered by EHR events. When a clinician opens a patient chart with recent imaging, Skolyn's CDS Hook fires and returns a card summarizing AI findings, including confidence scores, key image thumbnails, and direct links to the full AI report.

# CDS Hooks Discovery Endpoint
GET /cds-services

{
  "services": [{
    "id": "skolyn-imaging-ai",
    "hook": "patient-view",
    "title": "Skolyn AI Imaging Analysis",
    "description": "Surfaces AI analysis findings for recent imaging studies",
    "prefetch": {
      "patient": "Patient/{{context.patientId}}",
      "imagingStudies": "ImagingStudy?patient={{context.patientId}}&_sort=-date&_count=5"
    }
  }]
}

2.2 SMART on FHIR App (Embedded Viewer)

Skolyn provides a SMART on FHIR embedded application that launches within the EHR's app framework. When launched, the app displays a comprehensive AI analysis dashboard showing all processed studies for the current patient, interactive attention map overlays, temporal trend analysis, and one-click report generation. The SMART app supports both EHR launch (context from the EHR session) and standalone launch (user selects the patient).

2.3 Result Push via HL7v2 ORU

For EHR systems that do not support CDS Hooks or SMART apps, Skolyn delivers results via HL7v2 ORU^R01 messages through the institution's interface engine. The ORU message populates the "Results" section of the patient chart, appearing as a structured diagnostic report alongside traditional radiology reports. This integration pattern is compatible with virtually every EHR system and requires only standard HL7 interface configuration.

MEDITECH Expanse supports SMART on FHIR app embedding and HL7v2 ORU result delivery. Configure the SMART app via MEDITECH's Web Application Management tool. HL7v2 interfaces are configured through MEDITECH's Interface Management and support both MLLP and TCP socket connections with TLS encryption.

4. Critical Finding Alerts

When Skolyn detects a critical finding (e.g., pneumothorax, pulmonary embolism, intracranial hemorrhage, aortic dissection), the platform triggers an alert workflow that delivers notifications through multiple channels simultaneously:

  • EHR In-Basket/Message Center: Structured alert to the ordering provider with finding summary, confidence score, and link to full report.
  • PACS Worklist Priority: Study promoted to STAT priority in the radiologist's reading worklist.
  • SMS/Pager: Direct notification to the on-call radiologist with accession number and finding type.
  • Push Notification: Mobile app push notification for clinicians with Skolyn Mobile installed.

Alert acknowledgment is tracked end-to-end: Skolyn records the timestamp when each notification is sent, delivered, and acknowledged, creating an auditable chain of communication for regulatory compliance.

5. Data Flow Architecture

The EHR integration data flow proceeds as follows: A radiology order is placed in the EHR, generating an ORM^O01 message that is routed to Skolyn via the interface engine. When the modality completes the acquisition, the study is forwarded to Skolyn's DICOM Gateway. Skolyn processes the study using the appropriate AI module, generates findings, and delivers results through all configured channels (DICOM SR to PACS, ORU to EHR, FHIR DiagnosticReport to CDS Hooks). The clinician views results natively within the EHR without any additional login or context switch.
