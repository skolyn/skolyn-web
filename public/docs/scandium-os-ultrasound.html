1. Module Overview

Scandium OS is Skolyn's ultrasound analysis module, uniquely designed to handle the real-time, operator-dependent nature of ultrasonographic imaging. Named after Scandium (Sc, atomic number 21), the lightest transition metal, the module embodies lightweight, rapid-fire inference — processing individual frames at 30 FPS for real-time guidance and complete clip-based studies in under 10 seconds. Scandium OS supports 12 anatomical subtypes covering echocardiography, obstetric, thyroid, musculoskeletal, vascular, breast, abdominal, renal, hepatobiliary, point-of-care ultrasound (POCUS), pelvic, and testicular applications. The module's speckle-aware processing pipeline and temporal clip analysis distinguish it from other modality modules, addressing the unique noise characteristics and dynamic nature of ultrasound data.

3. Architecture

3.1 Speckle-Aware Processing

Ultrasound images are inherently noisy due to coherent scattering (speckle), which confounds traditional image analysis algorithms. Scandium OS employs a dedicated speckle-aware preprocessing pipeline that goes beyond simple noise reduction. The pipeline uses a learned speckle model (trained on paired raw RF/B-mode data from 500K frames) that decomposes each frame into a speckle-free structural component and a speckle texture component. The structural component is used for anatomical analysis, while the speckle texture is preserved for tissue characterization tasks where speckle patterns carry diagnostic information (e.g., differentiating fatty liver from normal parenchyma). This dual-representation approach preserves clinically relevant texture features that would be lost by aggressive denoising.

3.2 Temporal Clip Analysis

Many ultrasound findings are dynamic — cardiac wall motion, lung sliding, bowel peristalsis — and cannot be assessed from static images alone. Scandium OS processes ultrasound clips using a Video Transformer architecture (TimeSformer variant) that captures spatiotemporal patterns across frame sequences. The temporal analysis engine processes clips of up to 300 frames (10 seconds at 30 FPS), extracting motion features through divided space-time attention. For echocardiography, the temporal engine identifies end-diastolic and end-systolic frames, calculates ejection fraction via Simpson's biplane method, and performs global longitudinal strain (GLS) analysis by tracking myocardial speckle patterns across the cardiac cycle.

3.3 Real-Time Streaming Mode

Scandium OS supports a real-time streaming mode for point-of-care applications where the ultrasound probe is still in the operator's hand. In streaming mode, Scandium OS receives frames directly from the ultrasound machine via DICOM Secondary Capture or vendor-specific streaming APIs, processes each frame in under 33ms (30 FPS rate), and overlays AI guidance annotations (e.g., "lung sliding detected," "B-lines: 4 per intercostal space") on a companion display or directly on the ultrasound system's screen (for compatible vendors). This mode is particularly valuable for emergency and point-of-care settings where real-time AI guidance can improve diagnostic accuracy for non-specialist operators.

5. Operator Guidance

Scandium OS includes an image quality assessment module that evaluates each acquired image for diagnostic adequacy. The module scores images on a 1-5 scale based on depth of field, gain settings, anatomical completeness, and probe orientation. Low-quality images trigger automated feedback suggesting adjustments (e.g., "increase depth," "rotate probe clockwise," "insufficient visualization of right kidney lower pole"). This feature is particularly valuable for training sonographers and ensuring consistent image quality across operators with varying experience levels.
