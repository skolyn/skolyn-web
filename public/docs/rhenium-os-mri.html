1. Module Overview

Rhenium OS is Skolyn's dedicated MRI analysis module, engineered to process the full spectrum of clinical MRI examinations across neurological, musculoskeletal, cardiac, breast, abdominal, and pelvic applications. Named after the element Rhenium (Re, atomic number 75), the module embodies the rare metal's exceptional properties — high melting point, density, and catalytic capability — reflecting the module's ability to handle the most demanding computational challenges in multi-sequence, multi-planar MR analysis. Rhenium OS supports 14 anatomical subtypes, each powered by a specialized deep learning ensemble trained on millions of clinical MR examinations from 200+ academic medical centers worldwide.

3. Architecture

3.1 Cross-Attention Fusion Engine

The defining architectural innovation of Rhenium OS is the Cross-Attention Fusion (CAF) engine. Unlike traditional approaches that analyze each MR sequence independently, CAF jointly processes all available sequences for a given anatomical location through a multi-head cross-attention mechanism. The CAF engine learns inter-sequence relationships — for example, how a T2-bright lesion correlates with restricted diffusion on DWI and enhancement patterns on post-contrast T1 — producing a fused representation that captures diagnostic information invisible to single-sequence analysis. The CAF mechanism uses 16 attention heads across 6 transformer layers, processing tokenized sequence patches with positional embeddings that encode both spatial location and sequence identity.

3.2 Volumetric Analysis Pipeline

Rhenium OS provides quantitative volumetric analysis for key anatomical structures and pathological findings. The pipeline performs automatic atlas-based brain segmentation (143 regions following the Desikan-Killiany atlas), tumor volumetry with necrotic core, enhancing component, and peritumoral edema sub-volumes (per RANO criteria), and longitudinal comparison with prior studies for growth rate calculation. Volumetric measurements use sub-voxel interpolation for measurement precision of 0.1 mL.

5. Temporal Comparison Engine

Rhenium OS includes a sophisticated temporal comparison engine that automatically registers current studies with prior examinations using deformable image registration (voxelmorph-based). The engine identifies and quantifies changes in lesion size, signal characteristics, and new findings, presenting a side-by-side comparison with change-mapped overlays. Temporal analysis is critical for monitoring treatment response (RECIST 1.1, RANO criteria), tracking multiple sclerosis lesion burden, and assessing tumor progression versus pseudoprogression.
